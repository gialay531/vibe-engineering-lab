# Git Quick Reference

A practical reference for the Git and GitHub concepts used while building TPM Copilot.

## Core Workflow

```text
Edit → Save → Review → Commit → Push
```

- **Edit:** Make changes to a project file.
- **Save:** Write those changes to the computer.
- **Review:** Inspect what changed before recording it.
- **Commit:** Create a named checkpoint in the local Git history.
- **Push:** Send local commits to GitHub.

## Important Concepts

- **Repository:** A project and its recorded history.
- **Working directory:** The project files currently on your computer.
- **Commit:** A named checkpoint containing a specific set of changes.
- **Branch:** An independent line of work within a repository.
- **Remote:** An online copy of a repository, such as the copy on GitHub.
- **Origin:** Git’s conventional name for the primary remote repository.
- **Push:** Send local commits to a remote repository.
- **Pull:** Bring remote changes into the local repository.
- **Clone:** Create a local copy of an existing remote repository.
- **Merge:** Combine changes from one branch into another.
